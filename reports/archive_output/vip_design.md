# VIP Predictor 设计文档

## 概述

VIPPredictor 整合 FootballAI 所有模块的"最佳调参"到一个统一预测器，通过六层流水线实现从赔率到最终预测的完整链路。

## 选型理由

### 为什么是这6个组件

| 组件 | 文件 | 最优指标 | 在VIP中的角色 | 为什么选它 |
|------|------|----------|--------------|-----------|
| **v3.2 生产模型** | `football_balanced_production.joblib` | Acc=59.20%, Draw-F1=0.504 | L4: 基准概率推理 | 经过全量数据训练的Stacking集成(XGB+LGB+OE+Meta)，Draw-F1 0.504显著高于随机基线，是稳健的基线 |
| **λ融合** | `lambda_fusion.py::fuse_lambda()` | α=0.65/0.75/0.55 | L1: 桥接模型与庄家 | 三层自适应权重(常规/战术剧变/决赛)解决了单一λ源不可靠的问题 — 模型λ重历史规律，庄家λ含实时信息 |
| **陷阱检测 v3.1** | `bookmaker_trap_detector.py` | 16引擎+W_ambiguity | L2: 诱导识别 | 从8引擎扩展到16引擎(含大小球背离、反波胆防线)，覆盖浅盘大热→凯利背离→资金过热→波胆锁盘全链路诱导模式 |
| **陷阱→概率桥** | `trap_probability_bridge.py::apply_trap_correction()` | k=0.04/-0.03/-0.06 | L3: Score→ΔP | 四档分段映射: 安全<2.0→+0.04, 正常≤2.8→0, 警惕≤3.5→-0.03, 高危>3.5→-0.06 |
| **13维堆叠向量** | `stacking_vector.py::build_stacking_vector()` | 13维 | 模型特征构建 | λ、概率、陷阱、RP、战术、伤病 → 标准13维输入，是连接上层分析与底层模型的桥梁 |
| **RP+分段修正** | `odds_inverse_calibrator.py` | RP分级+进球分段 | L6: 比分降噪 | RP风控溢价分级(≤3常规/3~8轻度/>8重度) + 进球分段修正(≤1球×1.08/≤3球×0.97/>3球×0.88) |

## 六层流水线架构

```
match数据 {odds_h/d/a, asian_handicap, score_odds, tactical_shift, ...}
         │
    ┌────▼─────────────────────────────────────────┐
    │  L1: λ融合                                   │
    │  · 赔率比例法 → p_book → λ_book (二分搜索)    │
    │  · v3.2模型 → model_probs → λ_model (逆推)    │
    │  · fuse_lambda(α=0.65/0.75/0.55) → fusion_λ  │
    │  贡献: 桥接模型(历史规律)与庄家(实时信息)      │
    └────┬─────────────────────────────────────────┘
         │
    ┌────▼─────────────────────────────────────────┐
    │  L2: 陷阱检测                                 │
    │  · BookmakerTrapDetector.detect()             │
    │  · 16引擎并行: E1浅盘大热/E2降赔升水/...      │
    │    /E15大小球背离/E16反波胆防线               │
    │  · 复合评分 = Σ(w_i·F_i) × W_hist × W_tactic  │
    │  贡献: flag诱导操作，保护下游不被误导          │
    └────┬─────────────────────────────────────────┘
         │
    ┌────▼─────────────────────────────────────────┐
    │  L3: 陷阱→概率桥                              │
    │  · apply_trap_correction(p_h,p_d,p_a,trap)    │
    │  · score<2.0→Δ+0.04 | ≤2.8→0 | ≤3.5→-0.03 |  │
    │    >3.5→Δ-0.06                                │
    │  贡献: 将陷阱信号转化为概率修正量              │
    └────┬─────────────────────────────────────────┘
         │
    ┌────▼─────────────────────────────────────────┐
    │  L4: v3.2 生产模型推理                        │
    │  · 72维特征构建(默认值+可推导特征)             │
    │  · XGB + LGB + OE → Meta融合 → model_probs    │
    │  贡献: 稳健基线，Draw-F1=0.504                │
    └────┬─────────────────────────────────────────┘
         │
    ┌────▼─────────────────────────────────────────┐
    │  L5: 加权融合                                 │
    │  · final = model×0.65 + corrected×0.35        │
    │  · 归一化 → 最终1X2概率                       │
    │  贡献: 利用模型的稳定性 + 陷阱桥的实时修正     │
    └────┬─────────────────────────────────────────┘
         │
    ┌────▼─────────────────────────────────────────┐
    │  L6: RP降噪 + 比分输出                         │
    │  · λ → 泊松比分矩阵 (7×7=49比分)              │
    │  · RP风控溢价 → 下调冷门比分概率               │
    │  · 进球分段修正 → 小比分增/大比分降            │
    │  贡献: 消除庄家刻意放大冷门赔率造成的噪声      │
    └──────────────────────────────────────────────┘
```

## 各层贡献量化

### L1 λ融合 — 核心价值: 双源信息互补

庄家赔率经过3层抽水+利润率+资金扰动，单一逆向不可靠; 模型λ从历史数据学习规律但滞后于实时变化。融合后:
- 正常比赛(α=0.65): 65%模型 + 35%庄家 → 偏重稳定性
- 战术剧变(α=0.75): 75%模型 + 25%庄家 → 庄家λ对临时变动更敏感，降低权重
- 决赛(α=0.55): 55%模型 + 45%庄家 → 决赛异常值多，增加庄家实时信号权重

### L2 陷阱检测 — 核心价值: 识别庄家诱导

16引擎覆盖5大类诱导模式:
- **盘口异常**: E1浅盘大热 / E3升盘降水 / E4平半高水 / E6深盘诱杀
- **赔率矛盾**: E2降赔升水 / E7抽水异常 / E10深盘退热
- **资金信号**: E11资金单向过热
- **机构防御**: E8波胆防线 / E9凯利背离 / E15大小球背离 / E16反波胆防线
- **背景参照**: E13历史参照偏差 / E14反击威胁

### L3 陷阱桥 — 核心价值: score→ΔP 非线性映射

通过四档分段, 避免简单线性映射造成的过调:
- 轻度陷阱(≤2.8): 不做修正, 避免噪声放大
- 中度陷阱(≤3.5): 小幅下移3pp, 保守应对
- 高度陷阱(>3.5): 大幅下移6pp, 反映严重信号

### L4 v3.2模型 — 核心价值: 50万+场次训练的稳健基线

- XGBoost (5层depth, 500棵树): 强非线性, 擅长λ-crush/sigma-trap等高阶交互
- LightGBM (7层depth, 800棵树): 平局偏重(×1.2权重), 缓解H/A类不平衡
- OddsExpert (6层depth, 600棵树): 10维纯赔率特征, 抽水×1.3权重
- Meta模型: 21维子模型概率融合, 自适应权重组合

### L5 加权融合 — 核心价值: 互补权重分割

- 模型65%: 系统性规律, 不可被单场噪声推翻
- 陷阱桥35%: 实时修正, 不因历史概率忽略当前信号

这种分割确保: 正常比赛偏向模型稳定输出; 陷阱比赛陷阱桥有足够话语权。

### L6 RP降噪 — 核心价值: 比分层去庄家噪声

- RP>3 (轻度风控): 下调总进球≥4 且概率<5%的冷门比分概率
- 进球分段修正: ≤1球(+8%修正, 小球常被庄家过度低估) / ≤3球(-3%) / >3球(-12%, 大比分赔率虚高)
- 泊松矩阵(7×7=49比分): 仅保留≥0.5%概率比分, 过滤噪声

## 验证结果

### 葡萄牙 1-1 刚果 (国际友谊赛)

| 指标 | 数值 | 解读 |
|------|------|------|
| 欧赔 | H=1.27 D=5.60 A=11.0 | 极度一边倒，庄家定价H胜率≈75% |
| 陷阱评分 | **9.88** | 触发5个引擎，重度陷阱 |
| 模型预测 | H=0.772 D=0.200 A=0.027 | v3.2也认为大概率主胜 |
| 陷阱修正 | H=0.729 D=0.180 A=0.091 | 陷阱信号下调H 4.3pp |
| **VIP最终** | **H=0.757 D=0.193 A=0.050** | 加权融合输出 |
| 真实结果 | **1-1 平局** | 成功识别高风险 |

VIP预测器正确地将这场标记为"高风险陷阱"并给出建议规避，同时D=19.3%的平局概率显著高于市场隐含的D≈16.9%。

### 陷阱检测详情

| 引擎 | 置信度 | 方向 | 信号 |
|------|--------|------|------|
| E1 浅盘大热 | 0.90 | 下盘 | 理论让2.0球实际-1.5球 + 大热 |
| E9 凯利背离 | 0.68 | 庄家慷慨 | avg=1.67，过高定价 |
| E11 资金过热 | 0.89 | 反向 | 88%资金押主胜但赔率不动 |
| E8 波胆防线 | 0.80 | 锁盘 | 7个比分被锁(赔率>50) |
| E12 欧亚背离 | 0.68 | 诱下 | 欧赔75%但亚盘浅让 |

## 风险与局限

1. **72维特征不完整**: 实时预测无法获取全部72维特征(如otsm_*, drift_*, discussion_growth等)，使用默认值填充，精确度受限制。建议接入实时数据源补齐。
2. **极端赔率主导**: 当赔率极度一边倒(如1.27/5.60/11.0)时，即使陷阱修正也难以逆转方向，只能降权不能翻转。
3. **比分赔率依赖**: RP降噪需要比分赔率数据, 缺此数据时降为理论值(RP≈1)。
4. **国际友谊赛**: 联赛先验参数未覆盖友谊赛，使用全局默认值。

## 后续优化方向

1. **接入实时数据源**: 补齐72维特征中的动态字段
2. **自适应融合权重**: 根据陷阱置信度动态调整model/corrected权重比例
3. **联赛扩展**: 为友谊赛/杯赛建立专门的联赛先验
4. **比分校准**: 将比分矩阵与RP防线信号联合输出置信度
