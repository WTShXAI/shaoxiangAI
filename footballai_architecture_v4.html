🔀 五、专家协同调度 — 四类协作模式

模式 A：全栈预测模式（6专家并行 + 2工程保障）

触发条件：用户请求完整赛前预测分析
总调度：赵统筹 → 路由算法部
执行专家：季泊松(量化) ‖ 杜博弈(博弈) ‖ 荣合众(集成) ‖ 曾均衡(平局) ‖ 施时序(时序) ‖ 毕建模(统计) — 六路并行
工程保障：数定规(数据管线) + 钱代驾(代码落地) + 稳如山(部署)
融合策略：荣合众主导 → 加权投票 → 贝叶斯融合 → 毕建模统计验证 → 赵统筹把关输出

模式 B：赔率深度挖掘模式（博弈论域主导）

触发条件：赔率异常检测、庄家行为分析、诱盘识别
主导专家：杜博弈
辅助专家：季泊松(赔率漂移量化) + 毕建模(异常显著性检验)
串行流程：杜博弈诱盘检测 → 季泊松漂移分析 → 毕建模统计验证 → 输出诱盘报告

模式 C：平局/冷门攻坚模式（不平衡域主导）

触发条件：Draw-F1下降、冷门预测优化、代价敏感调整
主导专家：曾均衡
辅助专家：季泊松(泊松λ_H≈λ_A条件) + 荣合众(融合权重调整)
串行流程：曾均衡特征分析 → 季泊松概率验证 → 荣合众权重调优 → 毕建模效果检验

模式 D：系统自主迭代模式（全团联动）

触发条件：Acc跌幅>2pp · 特征漂移告警 · Draw-F1持续下降 · 定时周检
总协调：赵统筹
四阶段流程：
① 并行诊断：季泊松+杜博弈+荣合众+曾均衡+施时序+毕建模 → 各维度诊断报告
② 交叉评审：赵统筹汇总 → 识别交叉影响 → 拆解优化任务
③ 并行优化：各专家输出优化方案 → 毕建模统计验证
④ 工程落地：钱代驾+稳如山评估 → 灰度上线 → 全量
