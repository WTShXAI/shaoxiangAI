## 三、各模型诊断

### 3.1 v3.2 / v4.0 / v4.1 Stacking 模型（4个表现完全一致）

**架构**: LGB + XGB + OddsExpert + Heuristic + NN → meta_learner (LightGBM)

**评估**:
- ✅ 主胜识别强：13 场真实主胜命中 12 场 (92%)
- ✅ 客胜识别正常：3 场全部命中 (100%)
- ❌ **平局识别为零**：10 场平局全部漏判
- ⚠️ v4.0 → v4.1 升级在世界杯数据上**无任何效果提升**

**根因**: 模型 argmax 时 pD 永远 < max(pH, pA)。原因有二：
1. 平局赔率结构天然 pD < pH 或 pA（除非均衡赛）
2. Stacking 的 meta_learner 在训练数据上学习到"平局=少数类"，倾向压制 pD

**v4.1 的优化点（DrawExpert ×0.25 衰减 + 阈值 0.46）**: 在世界杯上无效果，因为 pD 从未达到 0.46 阈值。这些优化是面向联赛数据设计的（OOF Acc=62.43%），但世界杯样本分布偏移大。

### 3.2 DrawExpert v1（平局专家，2分类）

**结果**: pDraw 恒定输出 0.331（所有 26 场相同）
**评估**:
- ❌ **特征对齐失败** — 模型的 72 维特征名与我们的简化构造方式不匹配，导致模型输入几乎是常数
- ❌ 输出低于 best_threshold=0.344，所有场次判为胜负
- ⚠️ 即使在 OOF 数据上 best_f1=0.4265，世界杯样本上完全失效

**根因**: DrawExpert 训练时的特征工程依赖完整 72 维向量（含联赛风格嵌入、球队近况等），而回测时只有赔率无法完整重建。这印证了"特征管线一致性是硬约束"。

### 3.3 Neural Network (football_nn_20260616_125617.pth)

**结果**: 11.54% 准确率，几乎全部预测客胜 (23/26)
**评估**:
- ❌ **权重加载错误** — PyTorch state_dict 的 key 前缀与 FootballNN 类不匹配
- ❌ BatchNorm 在 eval 模式下未正确应用 running stats
- ⚠️ 即使加载正确，NN 单独使用价值有限（设计为 Stacking 子模型，非独立预测器）

**根因**: nn_state_dict 在 v4.1 模型 dict 中存储，但单独从 .pth 文件加载时 key 路径不对。需要从 v4.1 模型 dict 提取 `nn_state_dict` 而非独立 .pth。

## 五、核心结论

### 5.1 模型表现排名

| 排名 | 模型 | 价值定位 |
|------|------|---------|
| 🥇 | v3.2 / v4.0 / v4.1 Stacking | **主胜/客胜主力预测器** — 但平局完全失效 |
| 🥈 | D-Gate v4.9 规则层 | **平局补救层** — 模式 C 超热门翻车识别有效 |
| 🥉 | 陷阱检测 + risk_tag | **风控标记层** — 不直接预测，过滤高风险 |
| ❌ | DrawExpert v1 单独 | 无价值（特征对齐失败） |
| ❌ | NN 单独 | 无价值（权重加载错误） |
| ❌ | 多市场模型（让球/进球/OU） | 不预测1X2 |

### 5.2 关键诊断

1. **Stacking 模型平局 F1=0 是结构性问题，不是模型版本问题**
   - v3.2 → v4.1 升级在世界杯上零提升
   - 根因：模型在联赛分布上训练，世界杯样本分布偏移大
   - 解决：依赖 D-Gate 规则层补救（已实现）

2. **DrawExpert 集成在 Stacking 内部有效，单独使用失效**
   - 在 v4.1 Stacking 内：DrawExpert 信号×0.25 衰减后融入 meta_learner
   - 单独使用：特征对齐失败，输出恒定
   - 不建议独立评估，应作为 Stacking 子模块使用

3. **NN 必须从 v4.1 模型 dict 提取，不能独立加载**
   - 正确路径：`v41_model['nn_state_dict']` + `v41_model['scaler']`
   - 错误路径：独立加载 .pth 文件

4. **生产系统实际表现受 SSE 流响应延迟影响**
   - 26 场中 10 场 SSE 解析失败，需修复流式响应稳定性
   - 已解析的 16 场准确率 ~50%，平局 F1=0.2

### 5.3 建议改进方向

| 优先级 | 改进项 | 预期收益 |
|--------|--------|---------|
| P0 | 修复 SSE 流响应稳定性（10场解析失败） | 完整 26 场系统级回测 |
| P1 | 重训 Stacking 元模型时**强制包含平局样本权重** | 平局 F1 从 0 提升 |
| P1 | 扩充训练数据到 100+ 场世界杯/洲际杯 | 减少分布偏移 |
| P2 | 修复 NN 独立加载（从 v4.1 dict 提取） | 可独立评估 NN |
| P2 | 修复 DrawExpert 特征对齐 | 可独立评估 DE |
| P3 | 升级 Odds Add-On 获取真实赔率漂移特征 | 模型输入更丰富 |

### 6.3 关键观察

**为什么 v3.2/v4.0/v4.1 表现完全一致？**

三个版本的差异主要在：
- v3.2 → v4.0: 加入 DrawExpert 子模型
- v4.0 → v4.1: DrawExpert 衰减×0.25 + 阈值 0.46

但在 argmax 模式下，这些差异都不影响预测结果——因为 argmax 只看哪个概率最大，而 DrawExpert 的衰减和阈值都不改变 argmax 的判定。这就是为什么三个模型在世界杯上表现完全相同。

**这并不意味着 v4.1 升级无价值** — 在 OOF 训练数据上 v4.1 比 v3.2 有显著提升（62.43% vs 59.2%）。差异来自训练数据分布与世界杯样本分布的偏移。
