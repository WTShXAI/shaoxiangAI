## 一、瓶颈诊断

回测48场的核心问题分布：

| 问题 | 场次 | 占比 | 严重程度 |
|------|:--:|:--:|:--:|
| 攻击力低估（强队进球预测偏低） | ~12 | 25% | 🔴 P0 |
| 漏平局（超热门未识别平局） | ~8 | 17% | 🔴 P0 |
| 让2球不穿律误触发 | ~3 | 6% | 🟡 P1（已修） |
| 方向误判（爆冷） | ~6 | 13% | 🟡 P1 |
| 比分差1球 | ~12 | 25% | 🟢 P2 |

**核心矛盾**：系统知道谁能赢（方向67-72%），但**不知道能赢多少**（精确率12-42%）。比分偏差主因：**强队攻击力系统性低估 + 平局场景识别不足**。

## 二、P0 优先优化（投入小、回报大）

### 1. 超热门自动平局预警
**问题**：西班牙1-1沙特、葡萄牙1-1乌兹别克、比利时0-0伊朗 — 超热门(imp>85%)平局全部漏掉。
**方案**：`full_linkage_predictor.py` Priority Gate 新增强制检查：
```
if imp > 85% and ou_line <= 2.5:
    mode_c_auto = True  # 强制激活D-Gate模式C
    draw_prob *= 1.5    # 平局概率×1.5
```
**预期**：漏平局从8场→2-3场，平局召回率+40%。

### 2. 强队进球期望×1.3全局生效
**问题**：加拿大6-0(预1-0)、荷兰5-1(预1-0)、英格兰4-2(预1-0) — 世界杯进球率高于联赛。
**方案**：在Poisson λ计算中，对FIFA排名前10的球队自动×1.15因子：
```python
if fifa_rank <= 10:
    lam_strong *= 1.15  # 世界杯顶级队进球率高于联赛
```
**预期**：比分精确率+8-12pp，强队预测从1-0/2-0→2-1/3-1。

### 3. 历史相似场检索增强
**问题**：同样的让2球场景（塞内加尔vs伊拉克、库拉索vs科特迪瓦），有时穿盘有时走水，系统判断不准。
**方案**：`_vote_three_paths()` 的历史路径(C)已有雏形，升级为：
- ALL_RESULTS中同hcp±0.25 + 同OU±0.5的比赛自动检索
- 检索结果影响lambda：同方向3场以上穿盘→λ×1.2

## 三、P1 结构优化（投入中、持续收益）

### 4. 赛后学习闭环自动化
**当前**：`post_match_learner.py` 已建，但手动触发。
**升级**：backtest脚本完成后自动调 `learn_from_backtest()` → lessons.json → 下次预测D-Gate读取激活规则。
```
回测完成 → 自动learn → 更新lessons.json → D-Gate读取 → 规则生效
```

### 5. 半场修正集成预测
**当前**：`_half_time_adjust()` 函数完成，未集成主流程。
**升级**：API端点 `/api/half_time_adjust` → 赛中传入半场比分 → 返回修正预测。
实时场景：上半场结束→自动调半场修正→推送提醒。

### 6. 三路径投票权重自动调整
**当前**：A(模型)=0.6, B(D-Gate)=0.9, C(历史)=0.5 固定权重。
**升级**：基于回测表现动态调整 — 如果D-Gate连续3场错，自动降权→模型接管。
