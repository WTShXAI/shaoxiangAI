## 二、错误分析

### 场次2: 德国 1-1 巴拉圭 — **完全误判**

**预测逻辑链**:
1. 德国小组赛7-1库拉索 → 屠杀预警(massacre_warning)
2. Priority Gate 短路 → 跳过Chain 1-3, 锁定屠杀方向
3. 必发平局异常热(成交24.2% vs 理论18.6%) → 解读为「诱平陷阱」
4. 结论: 让胜 + 大球

**为什么失败**:
- 德国屠杀数据来自小组赛(7-1库拉索)，不适用淘汰赛
- 巴拉圭小组赛**2场零封**(0-0澳洲, 1-0土耳其)，场均GA=0.33 — 防守极强
- 必发平局热是**真实信号**，不是诱平陷阱 — 弱队防守好 → 市场准确预判了僵局
- 淘汰赛节奏保守，强队不敢全力进攻(怕被反击)

**根因**: 缺少「淘汰赛屠杀降级」和「必发平局异常热分类」机制

### 场次3: 荷兰 1-1 摩洛哥 — **方向正确但报告有矛盾**

**报告矛盾**: "方向：让平/让负（摩洛哥+0.5）" BUT "比分：首选 1-0（荷兰小胜）"
- 方向预测摩洛哥不败，比分预测荷兰1-0胜 → **自相矛盾**
- 实际上1-1平局才是正确的

**根因**: v5.18 hcp方向约束(让球方向过滤)错误移除了平局比分
- hcp=-0.5(主让) → v5.18过滤`diff≤0`的比分 → 所有平局/客胜被移除
- 候选池只剩主胜比分 → 1-0成为首选(与让负方向矛盾)

## 三、Bug修复清单 (v5.24)

### Bug 1 (P0): v5.18 hcp方向约束与平局预测冲突
**位置**: `taoge_strategy.py` L498-500
**修复**: 当primary方向含「平」时跳过方向过滤
```python
is_draw_verdict = ('平' in str(primary) and '让' not in str(primary)) or '平局' in str(primary)
if not is_draw_verdict:
    if is_home_give and diff <= 0: continue
    if not is_home_give and diff >= 0: continue
```
**影响**: 荷兰vs摩洛哥方向-比分矛盾 ✅已修复
