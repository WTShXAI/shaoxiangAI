## 三、P0级详细方案（立即执行，1-3天落地）

### P0-1: 代价敏感阈值分类器

**数学本质**: 当前H+0.02/D+0.10/A-0.04启发式不反映F1代价结构，D偏移不足以弥补argmax歧视。

**方案**: 
- Bayes最优 argmax_i(p_i − λ_i)
- λ_i从代价矩阵C(i,j)导出，C(D,¬D) > C(¬D,D)（平局漏判代价更高）
- 验证集grid search λ_i最大化Macro-F1
- 仅改决策边界，概率不变，与后续优化正交叠加

**收益**: D-F1 +0.02~0.03 | Macro-F1 +0.01~0.02 | LogLoss不变
**难度**: 低（改prediction_service.py一处决策逻辑）
**风险**: 低（阈值是外部参数，不改模型）

### P0-2: D-Gate Mode A梯度化+AND确认

**博弈论本质**: Mode A在imp 50-72%一刀切判平局，等于在庄家最自信的"主胜合理定价区"反向对赌。53%误判D→H说明庄家定价有效，系统跟有效市场对赌。

**方案**:
- **梯度化拆分**: imp 50-58%（弱热门）→ 不触发Mode A，走默认; imp 58-72%（中热门）→ 要求平赔<3.2 AND OU≤2.0双重证据才触发
- **Mode B→AND确认**: 均衡≠平局。Mode B不再独立启用(enabled=false维持)，改为Mode A的确认条件: spread<0.20且平赔[3.0,4.5]时，给Mode A额外+1分而非独立触发
- **S4∧S5矛盾惩罚**: 诱盘(S4:水位≥2.0) + 热门(S5:赔率1.20-2.00)逻辑矛盾→取max而非sum，减1分惩罚

**收益**: D-F1 0.5161→0.62~0.65 | Acc 42.3%→55~58%
**难度**: 低（改dgate_layer.py + drawgate_v53_rules.json）
**风险**: 低-中（回测验证26场，需确保不漏掉真平局）

### P0-3: λ融合去overround+贝叶斯调权

**数学本质**: 庄家赔率含overround(~5-7%)未去除=引入系统偏差。A+213强烈暗示融合拉偏向客胜。

**方案**:
1. 赔率去margin归一化再转λ: p_i = 1/odds_i / Σ(1/odds_j)，消除抽水偏差
2. 融合权重贝叶斯优化: p_fused = softmax(w_m·logit_m + w_b·logit_b)，grid search (w_m,w_b)最大化Macro-F1约束LogLoss<1.0
3. 当前w_m=0.55/w_b=0.45，预计最优在w_m∈[0.6,0.7]/w_b∈[0.3,0.4]

**收益**: 分布偏移减50-70% | D-F1 +0.01~0.02 | LogLoss −0.02~0.03
**难度**: 中（需修改unified_predictor.py融合逻辑）
**风险**: 低-中（去overround是教科书操作，调权需回测验证）

## 四、P1级详细方案（本周落地）

### P1-1: 温度标定+逐类偏置

**数学本质**: isotonic非参数化，D类阶梯平台压平pD>0.4区段使阈值不可达——校准改善LogLoss但损D-F1。

**方案**:
- p_i = softmax((logit_i + b_i) / T)
- T全局缩放 + b_i逐类偏置(3自由度)
- 保序保排名无平台，比isotonic更安全
- Grid search (T, b_H, b_D, b_A)最大化Macro-F1

**收益**: LogLoss −0.03~0.04 | D-F1 +0.005~0.015
**难度**: 低（在softmax前加温度和偏置）
**风险**: 低（保序性质，不破坏概率排名）

### P1-2: 特征有效性审计+域敏感隔离

**数据本质**: 80-100维中存在"域脆弱特征"——主客场特征(杯赛中立场失效)、联赛积分排名(跨联赛不可比)、历史交锋(不同联赛球队样本稀疏)。

**方案**:
1. 联赛holdout vs 世界杯26场做Permutation Importance + SHAP分布对比
2. 标记重要性漂移>阈值的特征为domain-fragile
3. 杯赛推断时对脆弱特征降权或置零(attention mask)
4. 重点审查draw_d_scorer 18维平局专精特征在杯赛是否仍有效

**收益**: Acc +2~3% | 降噪
**难度**: 低-中（审计工具已有，执行需1-2天）
**风险**: 中（26场样本统计功效弱，结论需bootstrap置信区间佐证）

### P1-3: D-Gate新增S7赔率变动信号

**博弈论本质**: 赔率变动反映庄家临场态度变化——平赔下降→庄家主动防平(+2)，上升→诱平(-1)。

**方案**:
- 新增S7: 平赔变动方向和幅度
- 平赔下降>5% → +2(庄家防平)
- 平赔上升>5% → -1(诱平信号)
- 需对接赔率历史数据（API支持实时赔率变动）

**收益**: 平局召回+5~8%
**难度**: 中（需赔率变动数据源+对接）
**风险**: 低（纯信号增强，不改现有逻辑）

## 五、P2级详细方案（本月推进）

### P2-1: DrawExpert重训+Focal Loss

**数学本质**: 临时阈值0.344无理论依据，L2未重训致正则化失配。交叉熵对低基率D天然不利。

**方案**:
1. Focal Loss + L2(λ=1e-3)重训DrawExpert
2. Youden J = TPR−FPR最大化求阈值，预期最优~0.28-0.32
3. 一维输出单独isotonic(1D安全)
4. α_D=1.5~2.0, γ=2.0 聚焦D边界hard examples

**收益**: D-F1 +0.03~0.05 | Macro-F1 +0.02~0.03
**难度**: 中（需重训，需验证集）
**风险**: 中（Focal Loss需调参，可能过度聚焦D导致H/A降幅超0.02守门线）

### P2-2: 平局定向数据增强

**数据本质**: 联赛平局稀疏(25%)，杯赛平局模式不同(保守战术、僵局)，模型对"杯赛型平局"特征空间覆盖不足。

**方案**:
1. 从联赛提取"类杯赛"子集: 赔率差<0.3的势均力敌场次、中立场、赛季末关键战
2. Cost-sensitive learning: 平局类损失权重×1.5
3. 特征空间SMOTE过采样平局样本，限制在"类杯赛"流形上合成

**收益**: 平局召回+8~12%
**难度**: 中（需数据工程+SMOTE实现）
**风险**: 中-高（合成样本若脱离杯赛流形会加剧偏移，需Wasserstein距离监控）

### P2-3: 小样本贝叶斯收缩

**数据本质**: 部分<5场球队，GA等统计特征点估计方差极大，直接输入=注入噪声。

**方案**:
1. Empirical Bayes多层收缩: 球队→联赛/区域→全局三级先验
2. 后验均值 = 样本估计与先验的精度加权组合，样本量越少收缩越强
3. <5场球队输出置信度降级标记，下游融合时降权
4. 先验锚点: FIFA排名 + Elo rating

**收益**: 小样本球队预测稳定性+15~20%
**难度**: 中（需新建收缩模块）
**风险**: 低-中（收缩过度抹平真实强弱差，需设收缩下限）

## 六、P3级详细方案（持续投入，长线根治）

### P3-1: 杯赛数据扩充+元学习适配

**数据本质**: 数据库无杯赛数据是根因性缺陷，26场世界杯不足以独立训练但足以few-shot适配。

**方案**:
1. 引入外部杯赛数据: 历届世界杯、欧洲杯、美洲杯、非洲杯
2. 跨联赛桥接: 欧战/世俱杯交锋记录建立实力映射
3. MAML式元学习: 联赛学通用表示，杯赛few-shot内循环适配
4. 规则层输出作为伪标签，半监督纳入无标注杯赛样本

**收益**: Acc+4~6%, 根因治理
**难度**: 高（数据对齐+元学习训练复杂度高）
**风险**: 高（外部数据口径不一致需对齐工程；元学习需大量实验验证）

### P3-2: D-Gate overround占比信号

**博弈论本质**: 平局抽水占比<30%→真防平(庄家认为平局有真实概率)，>50%→诱饵(庄家用平赔吸收资金)。

**方案**: 新增S8信号 = overround_D / total_overround
- <30% → +2(真防平)
- 30-50% → +1(中性)
- >50% → -2(诱饵)

**收益**: 平局精确度+3~5%
**难度**: 中高（需赔率原始数据精确计算overround分解）
**风险**: 中（依赖数据源精度）

## 九、风险控制铁律

1. **每项改动经回测实证验证** — 稳健调参，不赌
2. **守门规则**: D-F1不降 AND H/A各降幅<0.02
3. **永久禁用Beta校准** — 铁律不可违背
4. **分布偏移铁律**: OOF联赛提升≠世界杯提升，双域验证
5. **小样本保守铁律**: sample<5→保守，不激进
6. **一致性铁律**: 推荐报告7条校验自动执行
7. **代码产出先落沙箱** — 验证后再合入主线
