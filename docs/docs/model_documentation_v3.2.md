## 1. 项目概述

### 1.1 项目背景
FootballAI 是一个基于机器学习的足球比赛预测系统，目标是预测足球比赛的三个可能结果：
- **H** (Home win) - 主队胜
- **D** (Draw) - 平局
- **A** (Away win) - 客队胜

### 1.2 核心方法论

**核心理念：赔率 = 加密协议**

我们认为庄家赔率不是直接的定价信号，而是经过非线性变换的"加密产物"。预测的本质是逆向工程：
- 通过赔率漂移 (odds drift)
- 凯利指数 (Kelly index)
- 水位震荡 (odds movement)

剥离噪声，还原庄家风控的"锁定期"信号。

### 1.3 技术栈

| 组件 | 技术 |
|------|------|
| **机器学习** | LightGBM, XGBoost, Stacking, Neural Network |
| **特征工程** | 赔率特征 + 非赔率特征 (84维) |
| **后端** | FastAPI + uvicorn |
| **前端** | Vue 3.4 + Vite 6 |
| **数据** | SQLite + Parquet |
| **部署** | 本地开发环境 |

## 2. 核心问题摘要

### 🔴 问题1: 赔率传声筒问题 (Odds Echo Problem)

**现象**:
- 模型预测概率与赔率隐含概率高度相关 (相关系数 > 0.85)
- 非赔率特征重要性仅占 28.9%
- 模型没有产生独立的预测信号

**影响**:
- 预测缺乏附加值，基本等同于"赔率翻译"
- 难以捕捉赔率未反映的深层模式
- 对异常赔率（庄家故意误导）敏感

**当前缓解措施**:
- P3 特征工程：扩展非赔率特征到 84 维
- 增加球队近期状态、主客场表现、交锋历史等特征
- 正在验证效果（重训进行中）

---

### 🟡 问题2: 平局 (Draw) 预测不准确

**现象**:
- Draw F1 分数较低 (v3.2: 0.5040)
- 平局样本少且难以预测
- OddsExpert 对 Draw 的 F1 = 0.0300（几乎不预测平局）

**原因分析**:
1. **样本不平衡**: 平局占比约 22-25%，少于 H/A
2. **特征缺失**: 纯赔率特征无法有效区分平局
3. **模式复杂**: 平局往往由防守战术、天气、裁判等多因素决定

**已尝试方案**:
- ✅ Class weight 调整 (D: 1.1x)
- ✅ D门控规则 (Heuristic D > 0.30 且 H < 0.50 → 强制抬D)
- ✅ Override 路径改造 (阈值 0.40 → 0.70)
- 🔄 P3 新特征（预期改善）

**测试效果** (4场盲测):
- 巴西 vs 摩洛哥 (实际 D 1-1): **D门控命中** ✅
- 卡塔尔 vs 瑞士 (实际 D 1-1): 仍预测 A ❌

---

### 🟡 问题3: Stacking Meta-learner 性能不稳定

**现象**:
- Stacking Acc 有时低于基模型
- Meta-learner 容易过拟合 OOF 数据

**案例**:
- V5 初始版本: Stacking Acc = 52.82% (低于 LGB 的 57.64%)
- 原因: 时间切分不严格，有数据泄漏

**修复方案**:
- ✅ 采用严格时间切分 (pre-2023 训练, 2023+ OOF)
- ✅ 增加正则化 (reg_alpha=0.1, reg_lambda=0.1)
- ✅ 降低 class_weight 激进程度 (D: 3.0 → 1.1x)

**当前状态**:
- v3.2: Stacking Acc = 59.20% (高于基模型)
- Test Acc: 59.20% | Draw F1: 0.5040 | AUC: 0.8140

---

### 🟢 问题4: NeuralNet CUDA 兼容性问题

**现象**:
- `CUDA error: no kernel image is available for execution`
- RTX 5070 Ti 与 PyTorch CUDA 版本不匹配

**解决方案**:
- ✅ 增加 CUDA 可用性测试
- ✅ 自动降级到 CPU (虽慢但稳定)
- 🔄 未来考虑升级 PyTorch 或固定 CUDA 版本

---

### 🟡 问题5: 冷启动问题 (新球队/新联赛)

**现象**:
- 新球队历史数据少，特征坍缩
- 预测置信度低，容易误判

**解决方案**:
- ✅ 智能默认值 (基于联赛平均)
- ✅ 动态赔率权重 (冷启动时提高赔率依赖)
- 🔄 考虑迁移学习 (用相似球队初始化)

## 3. 模型架构

### 3.1 整体架构图

```
┌─────────────────────────────────────────────────────────┐
│                   预测请求 (match_id)                    │
└───────────────────┬─────────────────────────────────────┘
                    ▼
        ┌───────────────────────────────┐
        │     FeatureBuilder            │
        │  (特征工程 + 冷启动处理)      │
        └───────────┬───────────────────┘
                    ▼
        ┌───────────────────────────────┐
        │     OddsAnalyzer              │
        │  (赔率漂移 + 凯利指数)       │
        └───────────┬───────────────────┘
                    ▼
        ┌───────────────────────────────┐
        │     DegradationChain          │
        │  (三档降级: ML→赔率→先验)    │
        └───────────┬───────────────────┘
                    ▼
        ┌───────────────────────────────┐
        │     Fusion + Guard            │
        │  (多模型融合 + 37项检查)      │
        └───────────┬───────────────────┘
                    ▼
        ┌───────────────────────────────┐
        │     输出 (H/D/A 概率)         │
        └───────────────────────────────┘
```

### 3.2 集成模型组成 (v3.2)

| 模型 | 类型 | 特征 | 权重 | 专长 |
|------|------|------|------|------|
| **LightGBM** | 树模型 | 84维 | 1.0x | 综合预测 |
| **XGBoost** | 树模型 | 84维 | 1.0x | 防过拟合 |
| **OddsExpert** | 规则 | 赔率特征 | 1.5x | 赔率模式识别 |
| **Heuristic** | 规则 | 混合 | 1.0x | 平局专家 |
| **NeuralNet** | 神经网络 | 84维 | 1.0x | 非线性模式 |

**融合策略**: Stacking (meta-learner: LogisticRegression)

### 3.3 关键组件说明

#### FeatureBuilder
- 输入: match_id
- 输出: 84维特征向量
- 功能:
  - 赔率特征 (drift, kelly, spread, ratio)
  - 非赔率特征 (球队状态, 主客场, 交锋历史)
  - 冷启动处理 (智能默认值)

#### OddsAnalyzer
- 输入: 赔率数据
- 输出: 赔率衍生特征
- 功能:
  - 赔率漂移 (初赔 vs 即时赔率)
  - 凯利指数 (庄家真实看法)
  - 水位震荡 (异常波动检测)

#### DegradationChain (三档降级)
```
档位1: ML模型可用 → 使用完整集成预测
档位2: ML失败但赔率可用 → 使用 OddsExpert + Heuristic
档位3: 仅先验可用 → 使用联赛平均概率
```

#### PredictionGuard (37项检查)
- 概率和 = 1
- 无极端概率 (>0.95 或 <0.01)
- 特征范围合理
- 赔率一致性
- ...

## 7. 关键技术与方法

### 7.1 时间切分策略 (Time-based Split)

**问题**: 传统 K-fold CV 会导致时间泄漏 (用未来比赛预测过去)

**解决方案**:
```
训练集: 2012-2022 (pre-2023)
验证集: 2023+ (诚实OOF)
```

**效果**:
- 消除数据泄漏
- OOF 样本数从 3,603 → 8,631 (+139%)
- Draw F1 从 0.30 → 0.50 (+67%)

### 7.2 赔率漂移特征 (Odds Drift)

**定义**: 初赔 (opening odds) 与即时赔率 (closing odds) 的变化

**计算**:
```python
drift_H = (closing_odds_H - opening_odds_H) / opening_odds_H
```

**解读**:
- drift_H < 0: 庄家看好主队 (赔率下降)
- drift_H > 0: 庄家不看好主队 (赔率上升)
- |drift| > 0.10: 异常波动，可能有内幕消息

### 7.3 凯利指数 (Kelly Index)

**公式**:
```
Kelly_H = (p_H * odds_H - 1) / (odds_H - 1)
```
其中 p_H 是真实概率 (我们预测的概率)

**用途**:
- Kelly > 0.05: 有价值投注
- Kelly < 0: 不建议投注

### 7.4 降级链 (DegradationChain)

**设计理念**: 预测系统应该优雅降级，而不是硬失败

**三档降级**:
1. **ML档**: 所有模型可用 → 完整集成
2. **赔率档**: ML失败 → OddsExpert + Heuristic
3. **先验档**: 仅先验 → 联赛平均概率

**实际应用**:
- 新联赛 (无历史数据) → 自动降级到档位3
- 模型加载失败 → 自动降级到档位2
- 特征缺失 → 智能默认值 + 档位2

## 8. 当前挑战与诊断

### 8.1 诊断清单

如果模型性能下降，按以下顺序检查：

#### ✅ 数据问题
- [ ] 训练数据是否最新？(应包含 2025 年数据)
- [ ] 是否有异常赔率？(如 odds_H < 1.01)
- [ ] 时间切分是否正确？(不能用未来数据)
- [ ] 样本是否严重不平衡？(D 占比 < 20%?)

#### ✅ 模型问题
- [ ] Stacking 是否过拟合？(meta-learner 复杂度)
- [ ] Class weight 是否太激进？(导致预测偏移)
- [ ] 特征是否足够？(非赔率特征 < 40维?)
- [ ] 是否有数据泄漏？(特征包含未来信息)

#### ✅ 工程问题
- [ ] 预测服务是否正常？(FastAPI 日志)
- [ ] 特征工程是否一致？(训练 vs 预测)
- [ ] 模型文件是否损坏？(joblib 加载测试)
- [ ] GPU 是否可用？(NN 是否用 CUDA)

## 9. 未来优化方向

1. **深度学习模型升级**
   - 当前: NeuralNet (简单 MLP)
