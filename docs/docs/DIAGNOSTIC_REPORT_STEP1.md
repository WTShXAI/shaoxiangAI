## 四、当前架构分析

### 4.1 模型架构

```
输入 (19特征) 
    │
    ├─→ StandardScaler
    │
    ├─→ XGBoost 分类器 (权重 50%)
    │   ├─ 三分类 (Home/Draw/Away)
    │   ├─ 类别权重 (balanced × 平局1.15倍)
    │   ├─ 早停 (50轮)
    │   └─ Isotonic 概率校准
    │
    ├─→ Ridge 回归 (权重 30%)
    │   ├─ 预测净胜球
    │   └─ Softmax → 三分类概率
    │
    └─→ 启发式规则 (权重 20%)
        ├─ a1 × 0.4 (盘口信号)
        ├─ a2 × 0.5 (基本面)
        └─ rank_diff × 0.0005
        ↓
    加权投票 → 三分类概率
```

### 4.2 特征工程

- **19个特征** 分为8个模块（盘口、基本面、情绪、时空、裁判、套利、α因子A1-A6）
- 缺失值填充用默认值，无额外插值
- 异常值裁剪在99分位数
- 标准化用StandardScaler

### 4.3 数据采集链路

```
Football-Data.org v4 API (唯一数据源)
    ├─ /matches → matches表
    ├─ /competitions/{code}/standings → standings表 (未使用)
    ├─ /competitions/{code}/teams → teams表 (未使用)
    └─ /matches/{id}/odds → odds表 (几乎未使用)

↓ 特征计算
FeatureCalculator.calculate_all_features()
    ├─ 8大模块19+公式
    └─ → match_features表
```
