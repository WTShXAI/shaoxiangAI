# ADR-003: 全链路联动管道设计

**状态**: ✅ 已实施

**日期**: 2026-06-27

**决策者**: 架构团队

---

## 背景

最初系统中预测引擎各自独立运行（DrawGate、HCP、OU、Poisson），缺乏协同机制。预测结果之间可能相互矛盾，且没有统一的质量评估和降级策略。

## 决策

设计全链路联动管道 (Full Linkage Predictor)，统一集成 DrawGate、HCP、OU、Poisson 等多种预测引擎，通过联动矩阵 (LINKAGE_MATRIX) 实现跨信号协同决策。

## 选项对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **全链路联动管道** | 统一入口、协同决策、统一降级策略 | 代码量大 (`full_linkage_predictor.py` ~81KB) |
| 独立引擎 + 后聚合 | 模块独立、可单独部署 | 缺少跨信号协同、结果冲突 |
| 单一集成模型 | 简单 | 缺少领域知识注入 |

## 核心设计

```
Input → Preprocess → [DrawGate, HCP, OU, Poisson, Elo] 
     → LINKAGE_MATRIX 协同 → UnifiedPredictor → 输出
     → 降级策略 (L1/L2/L3)
```

## 后果

**正面**:
- 单一入口简化调用链
- 联动矩阵实现跨信号协同（如 HCP 信号修正 DrawGate 概率）
- 统一的三层降级策略（LLM → ML → 规则）

**负面**:
- 文件体积大，后续需要拆分
- 联动逻辑耦合度高
