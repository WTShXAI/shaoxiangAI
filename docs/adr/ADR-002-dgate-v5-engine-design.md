# ADR-002: D-Gate v5 引擎设计

**状态**: ✅ 已实施（当前为 v5.3）

**日期**: 2026-06-24

**决策者**: 算法团队

---

## 背景

预测引擎需要一种可配置、分层门控机制，在不同的比赛上下文中动态调整预测策略。v4.x 版本使用硬编码规则，难以维护和扩展。

## 决策

设计 D-Gate (Dynamic Gate) 五层引擎，基于 Elo 评分和上下文特征，为每个预测信号分配动态权重。引擎版本号与产品版本号分离管理（当前 D-Gate v5.3）。

## 选项对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **D-Gate 分层引擎** | 可配置信号权重、支持 A/B 测试、可观测 | 实现复杂度高 |
| 单一 ML 模型 | 简单直接 | 无上下文感知、难以调试 |
| 规则引擎 | 易理解、易维护 | 静态规则、无法适应动态场景 |

## 架构

```
输入 → Signal Extractor → Gate Controller → Weight Allocator → Ensemble → 输出
     ↓
  Context Analyzer (Elo, Tournament Phase, HCP)
```

## 后果

**正面**:
- D-F1 指标突破 0.31 (v5.3)
- 支持 A/B 测试不同信号权重组合
- 信号级别的可观测性和审计

**负面**:
- 层间依赖增加调试难度
- 需要独立的 D-Gate 配置管理
