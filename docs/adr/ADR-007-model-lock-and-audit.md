# ADR-007: ModelBridge v2.0 模型锁定与审计机制

**状态**: ✅ 已实施

**日期**: 2026-06-24

**决策者**: 算法团队

---

## 背景

早期版本中模型可以随意切换和回退，导致预测结果不可复现。同时，缺乏预测审计跟踪，无法追溯每个预测使用的模型版本和参数。

## 决策

设计 ModelBridge v2.0 单例模式，锁定生产模型为 `football_v4.1_production.joblib`，禁止运行时回退。所有预测输出包含审计字段 `_model`、`_version`、`_timestamp`，并存储预测快照（JSON）。

## 选项对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **ModelBridge v2.0 (锁定)** | 预测可复现、完整审计链 | 无法运行时切换模型 |
| 自由切换模型 | 灵活 | 预测不可复现、审计困难 |
| 版本控制 + 回退 | 有版本历史 | 回退可能导致生产事故 |

## 审计字段

```json
{
  "prediction": {"home": 0.45, "draw": 0.28, "away": 0.27},
  "audit": {
    "_model": "football_v4.1_production.joblib",
    "_version": "4.1.0",
    "_timestamp": "2026-06-28T12:00:00Z"
  }
}
```

## 后果

**正面**:
- 每次预测可精确复现
- 完整审计链支持合规需求
- 硬编码概率检测防止异常输出 (H=0.40/D=0.28/A=0.32)

**负面**:
- 模型更新需要重启服务
- 审计字段增加存储开销
