# ADR-001: Flask → FastAPI 迁移

**状态**: ✅ 已实施

**日期**: 2026-06-20

**决策者**: 架构团队

---

## 背景

项目最初基于 Flask 框架构建 API，后端结构为 `api/` 目录下零散的路由文件。随着系统复杂度提升（多智能体工作流、实时预测、A/B 测试等），Flask 的同步模型和缺乏类型验证能力成为瓶颈：
- 请求/响应无自动类型校验
- 无自动 OpenAPI 文档生成
- 同步 I/O 限制高并发场景性能

## 决策

将后端框架从 Flask 迁移到 FastAPI，目录结构从 `api/` 重构为 `backend/api/v1/endpoints/`。

## 选项对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **FastAPI** | 异步原生、Pydantic 校验、自动 OpenAPI、高性能 | 需要异步适配、学习成本 |
| Flask + Flask-RESTx | 迁移成本低 | 异步支持弱、生态老化 |
| Starlette | 轻量灵活 | 缺少电池、需要自行组装 |

## 后果

**正面**:
- 自动生成 `/docs` Swagger 文档
- Pydantic 模型提供编译时/运行时类型校验
- 异步端点支持高并发预测请求
- 模块化路由组织 (api/v1/endpoints/)

**负面**:
- 部分原有同步中间件需要替换
- 依赖注入模式 (Depends) 需要团队适应
