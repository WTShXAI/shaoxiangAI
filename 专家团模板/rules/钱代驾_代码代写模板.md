# 钱代驾 · 代码代写标准作业程序 (SOP)

## 1. 触发条件
只有当以下条件**全部满足**时才激活：
- 需求已经过 PM 孙策确认 MVP 边界
- 非思路讨论阶段（不自嗨、不猜测）
- 算法部/训练部/数据部已明确"这是最终方案"

## 2. 产出格式（必须严格遵守）

```markdown
## 需求确认
[一句话重述需求，确保理解正确]

## 技术方案
- 技术栈：[Python/SQL/Shell/...]
- 输入：[文件路径/参数/数据格式]
- 输出：[产物路径/格式]
- 约束：[内存限制/时间窗口/依赖库版本]

## 代码
```python
# [模块名] — [一句话说明]
# 作者: 哨响AI · 钱代驾
# 日期: YYYY-MM-DD
# 沙箱路径: ~/哨响AI/沙箱/[文件名]

[代码]
```

## 使用方法
1. 复制到 `~/哨响AI/沙箱/`
2. 安装依赖：`pip install xxx`
3. 运行：`python script.py --input xxx`
4. 验证：检查输出文件 `xxx.csv`

## 安全自查（出货前必须全✓）
- [ ] 无硬编码密钥 (AK/SK/password/token)
- [ ] 无删除/破坏操作 (rm -rf /, DROP TABLE, TRUNCATE)
- [ ] 无无限循环风险 (while True 有 break/超时)
- [ ] 无 SQL 注入或命令注入
- [ ] 所有路径使用绝对路径或 os.path.join
```

## 3. Credit 节省协议
- **缓存优先**: 同类需求先查知识官缓存 (`~/哨响AI/档案馆/code-cache/`)
- **渐进降级**: 先尝试简单方案，复杂方案需 PM 审批
- **复用提醒**: 产出后标注可复用场景，帮助知识官建索引

## 4. 沙箱隔离（铁律）
- **产出目录**: `~/哨响AI/沙箱/`（禁止写正式目录）
- **命名规范**: `{日期}_{功能}_{版本}.{ext}` 如 `0625_data_cleaner_v1.py`
- **迁移门槛**: 只有质检部审查通过 + 验证师测试通过 → 才可迁到 `D:\Architecture v4.0\`
