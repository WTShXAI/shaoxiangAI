# 安全卫士 · 代码审查标准作业程序 (SOP)

## 1. OWASP Top 10 速查（每次审查必过）
| # | 风险 | 检查要点 |
|---|------|---------|
| A01 | 访问控制失效 | 管理端点有无认证？`get_current_user()` 是否生效？ |
| A02 | 加密失效 | SECRET_KEY 是否硬编码？SSL 是否启用？ |
| A03 | 注入 | SQL/命令拼接？参数化查询？ |
| A04 | 不安全设计 | 速率限制？CSRF 保护？ |
| A05 | 安全配置错误 | .env 泄露？debug 模式开启？ |
| A06 | 脆弱组件 | 依赖库版本？已知 CVE？ |
| A07 | 认证失效 | JWT 过期？密码强度？ |
| A08 | 软件和数据完整性 | 文件上传校验？反序列化风险？ |
| A09 | 日志和监控失败 | 异常是否被吞？有没有告警机制？ |
| A10 | SSRF | URL 参数可被外部控制？ |

## 2. CodeBuddy 代码审查协议（钱代驾产出后执行）

### Step 1: 自动化扫描
把代码丢给 CodeBuddy 自审，prompt 模板：
```
你是安全审计专家。审查以下代码：
1. 是否有硬编码密钥 (AK/SK/password/token/API_KEY)
2. 是否有删除/破坏操作 (rm -rf /, DROP TABLE, TRUNCATE, os.system('del'))
3. 是否有无限循环风险 (while True 无 break, 递归无终止条件)
4. 是否有注入风险 (SQL拼接, os.system用户输入, eval)
5. 文件操作是否安全 (路径穿越, 权限过度)

代码：
[粘贴]
```

### Step 2: 风险评估
| 严重度 | 定义 | 响应 |
|--------|------|------|
| P0 致命 | 可导致数据丢失/系统崩溃/密钥泄露 | 立刻拦停，禁止部署 |
| P1 高危 | 可利用漏洞/绕过认证 | 修复后才能部署 |
| P2 中危 | 最佳实践违反/潜在风险 | 记录，排期修复 |
| P3 低危 | 代码风格/提示信息 | 建议修复 |

### Step 3: 审查报告模板
```markdown
## 安全审查报告
- 审查时间：[时间戳]
- 审查对象：[文件名/commit]
- 审查方法：OWASP Top 10 + CodeBuddy 自审

### 发现
| # | 严重度 | 位置 | 描述 | 修复建议 |
|---|--------|------|------|---------|
| 1 | P0 | L15 | 硬编码 API_KEY | 改为 os.getenv() |
| 2 | P2 | L42 | 异常被 except 吞掉 | 加 logger.error |

### 审查结论
- [ ] 通过 (无 P0/P1)
- [ ] 条件通过 (P1已修复, P2记录)
- [ ] 拒绝 (存在 P0)
```
