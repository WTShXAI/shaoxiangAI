# 智囊团 · 优化方案标准作业程序 (SOP)

## 1. 五视角交叉分析框架

任何回测问题，五位专家必须**独立出具**分析报告后再交叉验证。

| 视角 | 负责人 | 核心问题 |
|------|--------|---------|
| 全局统筹 | 何执策 | 这个问题的优先级？投入产出比？影响范围？ |
| 数学建模 | 毕正验 | 概率分布正常吗？损失函数适配吗？λ参数稳定吗？ |
| 博弈分析 | 渡庄生 | 庄家在这个赔率结构下的真实意图？D-Gate触发合理吗？ |
| 系统架构 | 费深谋 | 修复方案改多少文件？引入新依赖吗？能降级吗？ |
| 数据科学 | 贾证数 | 分布偏移了吗？样本量够吗？有data snooping吗？ |

## 2. 方案输出模板

```markdown
# [问题名称] - 优化方案

## 🏷️ 元数据
- 方案ID：PLAN-YYYYMMDD-[部门缩写]-[序号]
- 来源：回测发现 / 用户反馈 / 质检发现
- 影响范围：[N场比赛 / N个模式]
- 严重度：P0 / P1 / P2 / P3

## 📊 五视角诊断

### 数学建模师 · 毕正验
- 根因：[一句话]
- 证据：[具体数字/分布]
- 方案：[数学修正, 复杂度L/M/H]

### 博弈分析师 · 渡庄生
- 根因：[庄家视角一句话]
- 证据：[赔率数据/触发记录]
- 方案：[阈值调整/信号改造]

### 系统架构师 · 费深谋
- 方案A: 改动[N行/N文件], 风险[L/M/H], 回归[N]用例
- 方案B: 改动[N行/N文件], 风险[L/M/H], 回归[N]用例

### 数据科学家 · 贾证数
- 根因：[数据视角]
- 证据：[分布对比/样本量]
- 方案：[数据增强/重采样/特征筛选]

## 🎯 首席策略官综合方案

| 方案 | 预期收益 | 工作量 | 风险 | P级 | 推荐度 |
|------|---------|--------|------|-----|--------|
| A | +X% Draw-F1 | 1h | 低 | P1 | ⭐⭐⭐ |
| B | +Y% Acc | 4h | 中 | P2 | ⭐⭐ |
| C | 长期方案 | 2d | 高 | P3 | ⭐ |

## ✅ 建议执行团队
- 主执行: [算法部/训练部/数据部]
- 验证: [验证师]
- 审查: [质检部]
- 归档: [知识官]
```

## 3. P0-P3 分级标准

| 级别 | 定义 | 响应时间 | 示例 |
|------|------|---------|------|
| P0 | 上线阻塞，必须立即修复 | 即时 | 模型加载崩溃、API 500错误 |
| P1 | 高频高收益，优先修 | 24h | D-Gate模式误判率>30% |
| P2 | 有收益低风险，可排队 | 本周 | 新增特征、参数微调 |
| P3 | 锦上添花，无截止 | 本月 | 代码美化、文档完善 |

## 4. 方案执行跟踪
每份方案发布后，知识官自动建立跟踪卡片，记录：
- 方案状态：待执行 → 执行中 → 验证中 → 已上线 → 已归档
- 执行人、开始时间、完成时间
- 回测结果对比（优化前 vs 优化后）
